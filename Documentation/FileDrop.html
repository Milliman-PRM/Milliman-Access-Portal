File Drop User Guide

This user guide is intended to walk through how to manage file drops. Some of the content found in this user guide will pertain to the Client Administration tab within Milliman Access Portal (MAP).


File drop roles are set within a client. This means that the roles are set in the Client Administration tab of MAP. To set the roles for a user of the client, navigate to the Client Administration tab, filter to the client by clicking the client card, and then open the user card in the rightmost column. The user roles contain File drop user and File drop admin.

  • File drop user: A user who is capable of uploading, downloading, and deleting content (these permissions are set within a file drop), and can manage their own settings, but cannot view the file drop activity log or user permissions.
  • File drop admin: A user with the ability to set user permissions at the file drop level. A file drop admin can also view the activity log for the file drop as well as their settings.

Creating a File Drop

  1. Choose the client that you would like to create a file drop for
  2. Click "New File Drop"
  3. Fill in the file drop name and description (optional) and press "Add"

Editing a File Drop

  1. Navigate to the file drop to edit
  2. Click on the blue pencil icon on the file drop card in order to enter edit mode
  3. Click into the file drop name and/or description to edit
  4. Press the green check mark to save

Deleting a File Drop

  1. Navigate to the file drop to delete
  2. Click on the trash icon on the file drop card
  3. Confirm deletion of the file drop

Adding an Individual User to the File Drop

  1. Within MAP, navigate to the File Drop tab and select the "USER PERMISSIONS"
  2. Click the "Generate Credentials" ********check this
  3. Open a SFTP client outside of MAP
  4. Navigate to the site manager in the SFTP client
  5. Fill in the SFTP SSH file transfer protocol
  6. Fill in the host, port, and username information
  7. Copy the password generated by MAP and use in the SFTP client
  8. Connect to MAP

Creating a Permission Group

  1. Navigate to the file drop that you want to add a permission group to
  2. Click the pencil icon in the upper right to add User Permissions
  3. To add a user with their own permissions, click "Add User". To add a group of users with the same permissions, click "Add Group"
  4. Choose the user(s) to be added from the dropdowns and select which permissions they should receive between Download, Upload, and Delete
  5. Press "Save Changes"

Note that only users who have been given the role of "File Drop User" from the Client Administration page can be added to the User Permissions for a file drop. Also note that a user can only be assigned once per file drop. This means that if a user is a part of one permission group, they cannot be added to a different permission group for the same file drop.

Editing a Permission Group

  1. Navigate to the file drop that contains the permission group that you would like to edit
  2. Click on the edit icon in the upper right of the User Permissions form
  • Delete a user/permission group: Pressing the trash icon for a user or permission group will remove permissions for the user/group to the file drop
  • Add users/permission groups: Press "Add User" or "Add Group" to create more permission groups
  • Remove users from a specific permission group: Pressing the minus icon next to a user in a permission group will remove that user from the permission group
  • Edit permission group name: Edit a permission group name by clicking into the name and editing it

Setting Permissions for a Permission Group

  • Download: The ability to download files from the file drop
  • Upload: The ability to create folders and upload files to the file drop
  • Delete: The ability to delete folders and files from the file drop

MAP file drop features for uploading, downloading, and deleting folders and files are supported using Secure File Transfer Protocol (SFTP). Use of an SFTP client application such as FileZilla is the means to interact with the folders and files of a MAP file drop.

In a future release, MAP will also support folder and file operations directly in the user interface.

The activity log is available for file drop admins to see what actions users have performed in the file drop. You have the ability to refresh the page as needed. The most recent 30 days of activity is shown in reverse chronological order. Pressing the "Download All" button will download all activity for the file drop to your browser as a csv.

Definitions of Actions Shown in the Activity Log

SFTP account created:
A SFTP account was created

SFTP account deleted:
A SFTP account was deleted

SFTP account added to permission group:
A user was added to a permission group. A permission group can contain a group of users or be exclusively for single user

SFTP account removed from permission group:
A user was removed from a permission group. The affected user account and password are not destroyed but do lose access to the file drop until assigned to a new permission group

SFTP account account password generated:
A user generated a new password

SFTP authentication failed:
A connection attempt was blocked due to unsuccessful authentication or lack of authorization

File drop created:
The file drop was created

File drop deleted:
The file drop was deleted **********do we need this?

File drop updated:
The file drop name and/or description was updated

File drop permission group created:
A file drop permission group (for either a group of users or a single user) was created

File drop permission group deleted:
A file drop permission group (for either a group of users or a single user) was deleted. Affected user accounts and passwords are not destroyed but do lose access to the file drop until assigned to a new permission group

File drop permission group updated:
A file drop permission group was updated, changing the name or description of the group

SFTP directory created:
A SFTP directory was created

SFTP directory removed:
A SFTP directory was removed

SFTP file upload authorized:
A file was uploaded to the file drop

SFTP file download authorized:
A file was downloaded from the file drop

SFTP file delete authorized:
A file was deleted from the file drop

SFTP file or directory renamed:
A file or directory was renamed in the file drop

SFTP Connection:
This section provides connection details of MAP's SFTP server. Use these details to configure an SFTP Client application such as FileZilla

SFTP Credentials:
This section displays your SFTP login information for the currently displayed file drop. Please note that the username is case sensitive

The password can be reset at any time by clicking the button. The new password will be displayed and you can copy that to the clipboard using the provided button.  This is the only time that the password can be seen so please store it securely, such as in a password manager application

Notification Settings:
You can choose to receive email notifications each time certain file related events occur in the file drop. Use the toggle control to subscribe

Subscription changes apply only to the file drop currently displayed, and are effective immediately
