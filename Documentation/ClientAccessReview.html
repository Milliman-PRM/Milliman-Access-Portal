Sorting Clients

expand_more

When the page initially loads, the Client list will be sorted Chronologically Descending by review due date. To change the sort order of the Client list, click on the sort icon at the top of the Client column which lies next to the Client filter input. Clicking on the sort icon will cycle through the available sort orders.

Available Sort Orders:

  • Chronologically Descending: Clients with the most pressing Access Review due dates are sorted first. If a Client has sub-Clients, the entire Client group will be sorted according to the most pressing Access Review due date from the entire Client group.
  • Chronologically Ascending: Clients with the least pressing Access Review due dates are sorted first. IF a Client has sub-Clients, the entire Client group will be sorted according to the least pressing Access Review due date from the entire Client group.
  • Alphabetically Descending: Clients and sub-Clients are sorted alphabetically from A to Z by Client name.
  • Alphabetically Ascending: Clients and sub-Clients are sorted alphabetically from Z to A by Client name.

Performing a Client Access Review

expand_more

Periodic review of access to each Client is required by Client Admins to ensure that access to content is limited to only those who require access. To perform a Client Access Review, first click on the Client requiring review. After selecting a Client, a Summary of the selected Client will be displayed which displays information on the review due date, the last review, and assigned Client and Profit Center admins. To begin the Client Access Review, click on the BEGIN REVIEW button below the summary.

Client Access Review Steps:

  • Client Review:

    Review the following items:

    • Client Name
    • Client Code
    • Client Admins
    • Profit Center
    • Profit Center Admins
    • Approved Email Domain List
    • Email Address Exception List

    If any of these elements need to be updated before approval of the Client Access Review, they can be updated on the Client Admin page, with the exception of the Profit Center Admins which can be updated by submitting a request to map.support@milliman.com.

  • User Roles:

    Review all users assigned to the Client, their last login date, and the roles that they have been assigned for the Client. If any users need to be removed, or have their roles updated, this can be done on the Client Admin page.

  • Content Access:

    Review and approve content access for each Content Item for the selected Client. If any users access needs to be updated, this can be completed by a Content Access Admin for the selected Client on the Client Access Admin page. To move on to the next section of the review, each Content Item will need to be approved individually by checking the checkbox below the table listing Content Item membership.

  • File Drop Access:

    Review and approve user access and permissions to any File Drops that currently exist for the selected Client. If any user access or permissions need to be updated before approval of the Client Access Review, this can be done by a File Drop Admin for the selected Client on the File Drop page. To move on to the next section of the review, each File Drop will need to be approved individually by checking the checkbox below the table listing File Drop membership and roles.

  • Attestations:

    Review the displayed attestation language. By clicking COMPLETE REVIEW you are indicating that you have approved the Client Access Review and agree to the terms laid out in the attestation language. This action, and your attestation, will be recorded at this point, and the Client Access Review due date will be reset.

Note: Failure to complete a Client Access Review in a timely manner will lead to the suspension of content access until the Client Access Review is completed.

Exporting Client Access Review Summaries

expand_more

Client Admins have the ability to export the information found in a Client Access Review by clicking on the Download Client Access Summary button on the Client card, or by clicking on the Download Client Access Summary button within the view while a Client is selected. This action will download a ZIP file containing the following files:

  • Client_Summary.txt: This file contains summary information about the current Client.
  • Client_Access_Review_Summary_Metadata.txt: This file contains information about the currently exported Client Access Review Summary, such as the name of the exporter and the date the export was created.
  • User_Roles.csv: This file contains rows which list all users within the Client and their corresponding roles.
  • Content_Access.csv: This file contains rows which list all content items belonging to the Client with information about their selection groups and users.
  • File_Drop_Access.csv: This file contains rows which list all of the File Drops belonging to the Client with information about their permission groups and users.

Note: The information contained in this zip file represents a snapshot of the Client information at the time of export.
Viewing this information does not qualify as a Client Access Review.
