Managing Clients

expand_more

Creating a New Client

  1. Click the New Client card at the left, or the + icon to the right of the Filter Clients box
  2. Fill in the Client information form
  3. The Client Name and Profit Center fields are required
  4. Note that the username/email of a user of this client must match the Approved Email Domain List or Approved Email Address Exception List
  5. The user creating a new client is initially added to that client with Client Admin role
  6. Click the Create Client button to save the new Client with the entered information

Creating a Sub-Client

  1. Click the green plus sign on the Client card that you want to create a Sub-Client for
  2. Fill in the form, following the same steps as above for creating a new Client

Editing Client Information

  1. Identify the Client card to be managed in the left column
  2. Click on the blue pencil on the Client card, or on the pencil in the upper right of the form
  3. Click the desired information to update
  4. Note that the username/email of a user of this client must match the Approved Email Domain List or Approved Email Address Exception List
  5. Click Save Changes to update the client information

Requesting a Domain Limit Increase

  1. Send an email to MAP.Support@milliman.com
  2. In the email, include the name of the Client, the new domain limit, and a detailed reason for the necessity of the increase
  3. Once the request has been processed, you will be able to add new domains to the Approved Email Domain Limit up to the new limit

Deleting a Client

  1. Click on the red trash icon on the card of the Client to delete
  2. Click Delete in the popup message
  3. Enter the word "DELETE" and press the Delete button to confirm deletion

Managing Users in the Client

expand_more

Adding Users to a Client

  1. Click on the Client card to be managed
  2. Click the Add User card or the + icon near the top of the right column
  3. Enter the new user's email address in the Add User pop up
  4. Note that the user's email address must match the Approved Email Domain List or Approved Email Address Exception List of the Client
  5. Note that if the added user is new to MAP the user will immediately be sent a welcome email with a link to enable their account

Adjusting User Roles for the Client

  1. Click on a Client card to select the client to be managed. User cards for that Client will be displayed at the right

  3. Toggle the user's roles to turn off or on
  4. Note that a user's Admin role cannot be removed if they are the only admin for the Client

User Role Descriptions

  • Client Admin Enables the ability to create, edit, delete, and view Client information, manage the membership of Users of the Client, and manage each member User's roles for the Client.
  • Content Access Admin Enables the ability to grant or revoke access to specific content items by member Users that have the Content User role.
  • Content Publisher Enables the ability to publish content for the Client.
  • Content User Makes the User eligible to be granted access to Content of the Client. Note that a User with the Content Access Admin role still has to authorize this User to specific items of Content before Content can be accessed.
  • File Drop Admin Enables the ability to create, delete, and view File Drops of the Client, create and delete user groups for a File Drop, assign read, write, and delete permission to each user group, and manage user membership of each user group. This role also enables viewing and export of the activity log for the File Drop.
  • File Drop User Enables the ability to upload, download, delete, rename, and move files and folders in the File Drop, and to perform equivalent actions through the SFTP protocol, all subject to read, write, and delete permissions as granted by a User with File Drop Admin role. This role also enables the User to generate a new SFTP authentication password (passwords expire periodically).

Removing a User

  1. Click on a Client card to select the client to be managed
  2. Click on the red minus icon on the card of the user to be removed
  3. Click Remove to confirm
  4. Note that a user cannot be removed if they are the only Admin for the client
